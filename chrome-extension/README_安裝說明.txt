ATG熱能柔光 Chrome 自動注入版

安裝方式：
1. 打開 Chrome。
2. 到 chrome://extensions。
3. 開啟右上角「開發人員模式」。
4. 點「載入未封裝項目」。
5. 選擇資料夾：
   C:\Users\deamb\Desktop\ATG惡魔\chrome-extension

使用方式：
1. 進入 ATG 遊戲頁面。
2. 擴充功能會自動注入，不需要手動貼 Console 程式碼。
3. 右側會出現「熱能柔光 暴衝模型」面板。
4. 先進入遊戲房間，確認遊戲畫面可正常操作後，再點「啟動監測」。
5. 啟動後面板會依「刷新秒數」靜默實時監控，先等房間列表資料回來並顯示，再讀取完整詳情。
6. 點「立即刷新」可立刻更新目前房間資訊。
7. 點任一房間可選取，再點「單間」只更新該房間的詳細資料。
8. 背景掃描列表與更新詳情時會攔截自己送出的回傳資料給面板使用，不會改動遊戲頁面原本選中的機台詳情區。
9. 點房間卡片只會選取面板資料；只有按卡片上的「進入」按鈕才會送遊戲原本的空房進入/鎖台事件。
10. 「單轉成本」填 0 會用房間資料自動估算；填入你的單轉成本後，高爆點數會改用該數值換算。

如果還看到舊版畫面：
1. 舊版特徵是表格列只有「機台、報率、下注、狀態」，沒有熱能柔光卡片與免費遊戲轉數區塊。
2. 到 chrome://extensions 按此擴充功能的重新整理按鈕。
3. 重新整理遊戲頁面。
4. 如果你有開 Tampermonkey，請停用舊的「ATG Slot Room Rate Filter」，或改安裝新版 room-rate-filter.user.js。

數學評分邏輯：
- 不是單純看報率，而是用多因子模型計算「暴衝機率 %」。
- 分數來源包含：
  1. 目前/今日報率熱度。
  2. 前一小時相對近30天的短線動能。
  3. 近30天基準支撐。
  4. 今日、前一小時、近30天的連續性。
  5. 下注量樣本可信度，會把低樣本高趴往基準收斂。
  6. 免費遊戲未開轉數與前一/前二週期的比例。
  7. 單點尖峰、短線過熱、週期過長、非空房與資料不足的回落風險。
  8. 使用 logistic 曲線輸出百分比，避免線性加分把單一高報率直接排到最前。
  9. 依上述資料輸出預測類型，例如短線暴衝型、週期蓄能型、高熱延續型、低樣本試探型、單點尖峰回落型。
  10. 少下注量卻開高趴、短時間高趴但近30天基準不足，會判定為回落風險並降低暴衝機率。
  11. 免遊預估會用目前未開轉數、前一/前二轉數、短線動能、週期壓力、30日基準與回落風險，推估可能落點、開轉區間、尚差轉數與可信度。
  12. 高回報排名會綜合暴衝機率、目前報率、30日基準、短線動能、免遊距離、下注樣本與回落風險排序。
  13. 高爆點數是觀察估算值，排名只看房間資料，點數則用「單轉成本」換算；單轉成本為 0 時才用目前房間資料自動估算。

欄位說明：
- 暴衝%：熱能柔光模型計算出的暴衝機率百分比，越高代表目前資料條件越好。
- 報率：目前主要報率，優先使用今日資料。
- 下注：目前主要下注量，作為樣本可信度來源。
- 暴衝機率：房間下方的柔和流光能量條，顯示模型百分比。
- 動能：前一小時與30日基準的短線推升程度。
- 週期：未開轉數相對前一/前二週期的壓力。
- 回落：低樣本高趴、單點尖峰、過熱與資料不足帶來的折扣風險。
- 預測類型：依照短線動能、週期、熱度、樣本量分出的判斷類型。
- 今日：今日輸贏/下注計算的報率。
- 時：前一小時輸贏/下注計算的報率。
- 30日：近30天輸贏/下注計算的報率。
- 免費遊戲：遊戲回傳的未開轉數、前一轉數、前二轉數。
- 免遊預估：顯示「落點、開轉區間、尚差、可信度」。例如開轉區間 140-180 轉、尚差 20-60 轉，代表模型認為接下來較值得觀察的未開轉數範圍。
- 回報分：用來排名高回報房間的主分數，越高越靠前。
- 高爆點數：顯示估計點數區間、剩餘轉數、補樣本缺口與可信度，幫助判斷是否接近高爆條件。
- 單轉成本：你的單轉成本。填 0 代表自動估算；填固定值可避免房間累積下注量造成點數估算失真。
- 自動頁數 / 自動筆數：依視窗高度、已讀取房間數、每頁資料量與空房篩選狀態自動調整。
- 進入：只對空房啟用，按下後才會送出遊戲原本的進房事件；監控本身仍只讀。

注意：
- 這套邏輯只能用目前房間資訊做排序與風險評估，不能保證下一把一定出獎。
- 遊戲結果仍由伺服器與隨機/機率機制控制。
- 擴充功能只讀取頁面已回傳的房間資料，不會自動投注。
- 實時監控只讀取資料，不會主動進入房間；進入必須由你手動按「進入」。
